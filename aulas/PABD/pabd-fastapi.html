MÓDULO 01

Ambiente Virtual uv & GitHub

Antes de escrever qualquer linha de código, precisamos preparar o ambiente. Um ambiente virtual isola as dependências do projeto — pense nele como uma "caixinha" que guarda apenas as bibliotecas que você precisa.

⚡

O que é uv?

Gerenciador de pacotes e ambientes Python ultra-rápido escrito em Rust. Substitui pip + venv em um único comando.

📦

Por que usar?

10-100× mais rápido que pip. Resolve conflitos automaticamente e cria lockfiles reproduzíveis.

🔒

Isolamento

Cada projeto tem suas próprias dependências. Sem conflito entre versões de bibliotecas.

01

▾

MÓDULO 02

Primeiros passos com FastAPI

FastAPI é um framework web moderno para Python que permite criar APIs REST de forma rápida, com validação automática, documentação interativa e suporte a async nativo.

🚀

Ultra rápido

Performance comparável ao Node.js e Go graças ao Starlette e Pydantic.

📝

Docs automáticas

Interface Swagger gerada automaticamente em /docs.

✅

Validação built-in

Pydantic valida tipos, campos obrigatórios e formatos automaticamente.

01
Instalar FastAPI e Uvicorn
▾

O uvicorn é o servidor ASGI que roda a aplicação FastAPI.

BASH
# Adicionar dependências com uv
uv add fastapi uvicorn

# Verificar o que foi instalado
uv pip list
💡 DICA
O uv add além de instalar já registra a dependência no pyproject.toml e gera o lockfile uv.lock automaticamente.
02
Criar o arquivo main.py
▾

Crie o arquivo main.py na raiz do projeto. A aplicação começa com uma instância de FastAPI.

PYTHON — main.py
from fastapi import FastAPI
from pydantic import BaseModel
from typing import List
import uuid

# Instância da aplicação
app = FastAPI(title="Minha API", version="1.0.0")

# Modelo de dados com Pydantic
class Tarefa(BaseModel):
    titulo: str
    descricao: str
    concluida: bool = False

class TarefaResposta(Tarefa):
    id: str

# "Banco de dados" em memória (lista Python)
tarefas_db: List[TarefaResposta] = []

# Rota raiz
@app.get("/")
def raiz():
    return {"mensagem": "API funcionando! 🚀"}
03
Endpoint: Criar tarefa (POST)
▾
POST /tarefas

Recebe dados no corpo da requisição (JSON), valida com Pydantic e salva na lista.

PYTHON — adicionar em main.py
@app.post("/tarefas", response_model=TarefaResposta, status_code=201)
def criar_tarefa(tarefa: Tarefa):
    # Gerar ID único
    nova = TarefaResposta(
        id=str(uuid.uuid4()),
        titulo=tarefa.titulo,
        descricao=tarefa.descricao,
        concluida=tarefa.concluida
    )
    tarefas_db.append(nova)
    return nova
📌 CONCEITO
O decorador @app.post registra a função como handler do método HTTP POST. O parâmetro response_model define o formato da resposta, e status_code=201 retorna "Created".

Conectando ao Banco de Dados

Agora vamos persistir os dados em um banco de dados real. Usaremos o SQLite (arquivo local, perfeito para aprendizado) com SQL puro — sem ORM.

🗄️

SQLite

Banco de dados em arquivo único. Zero configuração, incluído no Python.

🔌

sqlite3

Módulo nativo do Python para conectar e executar SQL no SQLite.

📜

SQL puro

Você controla cada query — sem mágica de ORM. Aprenda o que realmente acontece.

02
Criar database.py — conexão e tabela
▾
PYTHON — database.py
import sqlite3
from contextlib import contextmanager

DATABASE_URL = "tarefas.db"

def criar_tabelas():
    """Cria a tabela se não existir."""
    with sqlite3.connect(DATABASE_URL) as conn:
        conn.cursor().execute("""
            CREATE TABLE IF NOT EXISTS tarefas (
                id        TEXT PRIMARY KEY,
                titulo    TEXT NOT NULL,
                descricao TEXT NOT NULL,
                concluida INTEGER DEFAULT 0
            )
        """)
        conn.commit()

@contextmanager
def get_conn():
    """Context manager para conexão segura."""
    conn = sqlite3.connect(DATABASE_URL)
    conn.row_factory = sqlite3.Row  # acesso por nome de coluna
    try:
        yield conn
    finally:
        conn.close()
📌 CONCEITO
row_factory = sqlite3.Row permite acessar os dados pelo nome da coluna (ex: row["titulo"]) ao invés de índice numérico. O contextmanager garante que a conexão seja sempre fechada, mesmo em caso de erro.
03
Criar models.py — schemas Pydantic
▾
PYTHON — models.py
from pydantic import BaseModel

class TarefaEntrada(BaseModel):
    """Dados que o cliente envia."""
    titulo: str
    descricao: str
    concluida: bool = False

class Tarefa(TarefaEntrada):
    """Dados que a API retorna (inclui id)."""
    id: str

    class Config:
        from_attributes = True  # leitura de objetos/dicts
04
Atualizar main.py com banco de dados
▾
POST /tarefas — persiste no SQLite
GET /tarefas — lê do SQLite
PYTHON — main.py completo com banco
from fastapi import FastAPI, HTTPException
from typing import List
import uuid

from database import get_conn, criar_tabelas
from models import Tarefa, TarefaEntrada

app = FastAPI(title="Minha API com Banco", version="2.0.0")

# Criar tabelas ao iniciar a aplicação
criar_tabelas()


@app.post("/tarefas", response_model=Tarefa, status_code=201)
def criar_tarefa(dados: TarefaEntrada):
    tarefa_id = str(uuid.uuid4())

    with get_conn() as conn:
        conn.execute(
            """INSERT INTO tarefas (id, titulo, descricao, concluida)
               VALUES (?, ?, ?, ?)""",
            (tarefa_id, dados.titulo, dados.descricao, int(dados.concluida))
        )
        conn.commit()

    return Tarefa(
        id=tarefa_id,
        titulo=dados.titulo,
        descricao=dados.descricao,
        concluida=dados.concluida
    )


@app.get("/tarefas", response_model=List[Tarefa])
def listar_tarefas():
    with get_conn() as conn:
        rows = conn.execute(
            "SELECT id, titulo, descricao, concluida FROM tarefas"
        ).fetchall()

    return [
        Tarefa(
            id=row["id"],
            titulo=row["titulo"],
            descricao=row["descricao"],
            concluida=bool(row["concluida"])
        )
        for row in rows
    ]
⚠️ ATENÇÃO
Sempre use parâmetros ? nas queries SQL — nunca concatene strings! Isso evita ataques de SQL Injection.
